2. Format: {"reasoning": "<brief thought process>", "focus": "piece" | "global" | "geometry", "identifier": "<square, rank, file, or null>", "target_class": "<ClassName>"}
3. POLARITY (CRITICAL): Pay exact attention to the words "advantage" and "disadvantage". If the user asks about a disadvantage, `target_class` MUST be a disadvantage class (e.g., `WhiteDisadvantage`, `ExposedKing`). NEVER output an advantage class for a disadvantage query!

</strict_rules>
